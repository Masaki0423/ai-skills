# trending-ai-topics

実行時点のAIトレンドを収集 → 3〜5件に厳選 → Slackに投稿する。

## 引数

- `--dry-run` : Slack投稿せず、整形済みメッセージの表示のみで終了
- `--count N` : 投稿するトピック数（デフォルト: 3件）
- `--digest` : 話題ごとの個別投稿ではなく、全話題を1メッセージにまとめて投稿

## 手順

### 1. トレンド収集

WebSearch で以下の観点を並列で検索する。いずれも**直近数日以内**の情報に限定すること（検索クエリに期間指定や「today」「this week」を含める）:

- 生成AI・LLMの新モデル/新機能リリース
- AIエージェント・自動化ツールの話題
- 開発者向けAIツール（コーディング支援等）のアップデート
- 一般ユーザー向けAIサービスの話題・便利な使い方（非エンジニアでも試せるもの）
- AI業界の動向（投資・規制・企業導入・市場トレンド）
- 日本語圏で話題になっているAI関連の記事・議論

### 2. トピック選定

収集結果から、以下の**3つの読者層をすべてカバー**するように選ぶ。デフォルトは各層1件・計3件（`--count` 指定時は増減。個別投稿がチャンネルを占有しないよう、デフォルトは控えめにする）:

- 🔰 **これからAIを触る人向け**: 普段使いのサービス（ChatGPT等）の変化、非エンジニアでも今日試せるもの。専門用語を使わず説明できるネタ
- 🛠 **開発者向け**: モデルリリース、API・開発ツールのアップデート、セキュリティ。スペックや価格など具体的な数字があるもの
- 🏢 **業界動向**: 投資・規制・企業導入・市場トレンド。「仕事の風向き」として読めるもの

共通の基準:

- **URLがある**: 一次ソースまたは信頼できる記事のリンクを必ず1つ添付。検証していないURLを組み立てない
- **鮮度**: 実行時点から見て新しいもの優先。1週間以上前のものは原則除外
- 重複や同一ネタの続報は1件にまとめる

### 3. メッセージ整形

**1話題 = 1メッセージ**で投稿する。各メッセージがスレッドの起点になり、話題単位で会話が分岐できる。

まずヘッダーを1件:

```
:robot_face: いま話題のAIトピック（YYYY-MM-DD）
この下に3件流します。気になった話題はスレやスタンプで反応どうぞ :eyes:
```

続けて話題ごとに1メッセージ（カテゴリ絵文字を先頭に付ける）:

```
:beginner: *<トピック名>*
<1〜2行の要約。専門用語を避け、何がうれしいかを書く>
→ <URL>
```

- カテゴリ絵文字: `:beginner:`（初学者向け） / `:hammer_and_wrench:`（開発者向け） / `:office:`（業界動向）
- 1メッセージ3〜5行。1メッセージに複数話題を入れない
- どれか1件に軽い問いかけを添えてよい（全件に付けるとしつこい）
- 憶測や未確認情報を断定調で書かない
- 要約のトーンは読者層に合わせる（初学者向けは平易に、開発者向けは具体的に）
- `--digest` 時は旧形式（全話題を1メッセージに結合、セクション見出し付き）で整形する

### 4. Slack投稿

環境変数 `SLACK_WEBHOOK_URL` から Incoming Webhook URL を取得し、ヘッダー → 初学者向け → 開発者向け → 業界動向 の順に1件ずつ投稿する。

```bash
for msg in "${MESSAGES[@]}"; do
  curl -sf -X POST -H 'Content-Type: application/json' \
    --data "$(jq -Rns --arg t "$msg" '{text: $t}')" \
    "$SLACK_WEBHOOK_URL"
  sleep 1  # Incoming Webhookのレート制限は約1件/秒
done
```

- 投稿前に必ず全メッセージをログに出す（プレビュー）
- `--dry-run` 時はここで終了し、curl を実行しない
- `SLACK_WEBHOOK_URL` が未設定なら投稿せず、その旨を伝えて dry-run 扱いで終了
- Webhook URL 自体を出力・ログ・コミットしない

> 補足: Incoming Webhook はレスポンスに `ts` を返さないため、「親メッセージのスレッドに各話題をぶら下げる」構成は取れない。必要になったら Slack App（bot token + `chat.postMessage`）に移行する。

## 禁止事項

- 会社名・案件名・顧客情報に触れない（一般公開情報のトレンドのみ扱う）
- 業務利用を推奨する表現をしない。「個人学習・情報交換」のスコープを守る
- 特定ツールの導入を組織に迫るようなトーンにしない
